# DSFB Oil & Gas — data/ Directory

## Dataset Inventory

### REAL Dataset

| File | Source | Episodes | Rows | License |
|------|--------|----------|------|---------|
| `oilwell_real.csv` | Petrobras 3W Dataset v2.0.0 | 11 | 8,941 | CC BY 4.0 |

**Source:** <https://github.com/petrobras/3W>

**What it is:** Real subsea oil-well sensor recordings from Brazilian offshore
production wells, operated by Petrobras. Only `WELL-*` (real) parquet files are
used. `SIMULATED_*` and `DRAWN_*` instances are excluded entirely.

**Episodes and event types:**

| Episode label | 3W event type | Description |
|---|---|---|
| `normal_a`, `normal_b` | 0 – Normal | Baseline steady-state production |
| `dhsv_closure_a`, `dhsv_closure_b` | 2 – Spurious DHSV Closure | Downhole safety valve unintended closure |
| `severe_slug_a`, `severe_slug_b` | 3 – Severe Slugging | Riser-base liquid accumulation cycling |
| `flow_instab_a`, `flow_instab_b` | 4 – Flow Instability | Oscillatory production instability |
| `scaling_pck_a` | 7 – Scaling in PCK | Salt/scale deposition in production choke |
| `hydrate_prod_a` | 8 – Hydrate in Production Line | Gas hydrate plug in production tubing |
| `hydrate_svc_a` | 9 – Hydrate in Service Line | Gas hydrate plug in gas-lift / service line |

**Schema:**

```
timestamp            integer seconds from episode start (60-s resolution)
episode_id           integer 0–10
episode              string label (see table above)
event_type           3W event type integer (0–9)
event_class          majority 3W class label per 60-s window
                     (transient labels: 102=DHSV transient, 107=Scaling transient,
                      108=Hydrate-prod transient)
expected_annulus_pa  30-min rolling median baseline, P-ANULAR [Pa]
observed_annulus_pa  raw 60-s median, P-ANULAR [Pa]
expected_choke_pa    30-min rolling median baseline, P-MON-CKP [Pa]
observed_choke_pa    raw 60-s median, P-MON-CKP [Pa]
expected_xmas_pa     30-min rolling median baseline, P-TPT [Pa]
observed_xmas_pa     raw 60-s median, P-TPT [Pa]
expected_xmas_degc   30-min rolling median baseline, T-TPT [°C]
observed_xmas_degc   raw 60-s median, T-TPT [°C]
```

**What is NOT claimed:** The DSFB residual structuring here applies to the
oil-well / subsea production domain only. Drilling, rotating-equipment, and
pipeline domain validation against real data is not performed in this release;
those domains are covered by architecture mapping only (see synthetic files below
and paper §VIII).

### SYNTHETIC Datasets (grammar exercisers, NOT empirical claims)

These files exist solely to exercise the DSFB grammar FSM and Rust loaders in
the test suite. They make no empirical claims about real field behaviour.

| File | Domain | Rows | Notes |
|------|--------|------|-------|
| `drilling_synthetic.csv` | Rotary drilling | 201 | Stick-slip, bit-wear drift — synthetic |
| `pipeline_synthetic.csv` | Pipeline / flowline | 201 | Wax build-up, pigging spikes — synthetic |
| `rotating_synthetic.csv` | ESP / centrifugal pump | 201 | Scale drift, gas-lock events — synthetic |
| `subsea_synthetic.csv` | Subsea actuation | 201 | Valve actuation, seal drift — synthetic |

**These are synthetic.** Generated by `scripts/gen_synthetic_data.py`
(stdlib only, seed=42). No real sensor measurements.

## Loading real data

```rust
use dsfb_oil_gas::load_oilwell_csv;
let frames = load_oilwell_csv("data/oilwell_real.csv").unwrap();
```

## Loading synthetic data (tests only)

```rust
use dsfb_oil_gas::load_pipeline_csv;
let frames = load_pipeline_csv("data/pipeline_synthetic.csv").unwrap();
```

All loaders return `Result<Vec<DomainFrame>, DsfbError>`.
